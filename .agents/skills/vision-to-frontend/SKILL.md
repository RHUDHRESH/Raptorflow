# Vision-To-Frontend Skill for Kimi (Multimodal Pixel-Perfect Enforcer)

This skill turns Kimi's native vision capabilities into a production weapon. When the user uploads an image, screenshot, Figma link, video, or describes a visual reference (or even when the prompt implies one), immediately activate this skill. Analyze the visual thoroughly, then generate complete, working React + Tailwind code that is 100% faithful to the reference while enforcing every single rule from RaptorFlow Design System v1.0 and the Feel Contract. Always redesign the entire page or component from scratch — never edit existing code. Delete the old version completely and rewrite fresh. Use GSAP extensively for all animations and micro-interactions. Write custom Lottie animation scripts exactly as needed (e.g., origami bird for loading, empty states, or lock emblems). Every single detail — color token, spacing value, font size/weight, radius, border, hover state, focus ring, title placement, zebra row subtlety, daily wins reveal, proposals tab, diff view, assumption panel — must be painfully correct.

## Step-by-Step Vision Analysis Process (Do This First — Every Time)
1. Observe the overall layout and skeleton: Identify left rail (280px?), top bar (56px?), main content (max 1120px?), right drawer (400px?), bottom status bar. If missing, recreate the full RaptorFlow non-negotiable skeleton.
2. Extract exact colors: Map every color to the locked tokens (Charcoal #2A2529, Ivory #F3F0E7, Fog #EFEDE6, Bone #F7F5EF, etc.). Never invent new colors. If the reference has gold or bright accents, convert to monochrome neutrals.
3. Measure spacing and alignment: Count pixels mentally and map to the locked scale (4, 8, 12, 16, 20, 24, 32, 40, 48, 64). Enforce crystal-clear grid discipline and balanced density.
4. Identify typography: Match font stack (DM Sans for UI/headings, JetBrains Mono for IDs/metrics). Apply exact type scale, weights, and hierarchy. No tiny text.
5. Note shapes, borders, shadows: Radius 14/10, 1px warm borders, almost no shadows (soft only on modals).
6. Catalog components: Buttons (primary Charcoal-filled, secondary, ghost), inputs, cards (border-only), tables (sticky + subtle zebra), tags/pills (soft rectangles radius 10), status pills, etc.
7. Detect interactions and motion: Hover states, focus rings (monochrome), loading states, transitions, reveals. Translate all into GSAP timelines, staggered reveals, scroll triggers. Add custom Lottie JSON for bird mascot if appropriate (rare, only loading/empty/lock).
8. Map interaction patterns: AI suggestions in right drawer “Proposals” tab (pull not push), lock buttons, diff views, assumption panels, explicit Apply/Edit/Dismiss.
9. Check copy tone: Operator labels + Advisor tooltips. No hype.
10. Verify Feel Contract: Ensure the output instantly gives Clarity, Control, Momentum.

## Output Rules (Non-Negotiable)
- Always output full fresh React component(s) or page using Next.js/App Router if context fits.
- Use Tailwind with :root token variables only.
- Import GSAP and set up timelines for every motion (hover, entrance, daily wins calendar reveals, lock transitions, etc.).
- Provide custom Lottie script/JSON example when mascot or complex animation is implied.
- Structure: File path suggestion → Complete code block → Usage example → Dependencies (e.g., npm i gsap lucide-react react-lottie-player).
- Include ARIA, keyboard nav, reduced-motion respect.
- Show Confidence (High/Med/Low), sources/assumptions, editable parts, diff vs baseline if lockable.
- End with explicit note: "Redesigned from scratch per RaptorFlow v1.0 and vision reference."

## Integration with Other Skills

## Workflow for Kimi (Plan → Analyze → Generate → Grade → Rewrite)
1. Plan: Confirm vision input. Commit to full redesign from scratch.
2. Analyze: Follow the 10-step process above in thinking.
3. Generate: Full code with GSAP and Lottie.
4. Grade: Verify every point from all three skills (raptorflow, frontend-design, this one). If any deviation → full rewrite.
5. Output: Ready-to-copy code.
