## The 6-Step Systematic Debugging Workflow (Mandatory — Never Skip a Step)
1. **Plan & Reproduce** — List exact symptoms. Use playwright + browserbase to reproduce in clean environment. Take screenshot/video via MCP. Note RaptorFlow rule violated (e.g., wrong token, GSAP not cleaned, layout not 280px rail).
2. **Isolate** — Use chrome-devtools MCP to inspect DOM, network, console, GSAP timelines, React DevTools. Narrow to one file/component/route.
3. **Fix** — Redesign the entire component/page from scratch using raptorflow-design-vibe + all other skills. Apply performance-optimization, frontend-animations, form-handling, nextjs-app-router rules.
4. **Verify** — Run full test suite (frontend-testing skill). Lighthouse audit via MCP. Visual regression with playwright. GSAP timeline check. Lottie bird test if present. Lock/Draft flow test.
5. **Prevent** — Add new test case in frontend-testing. Add comment // MCP-enforced: RaptorFlow token + GSAP cleanup. Commit via github MCP with detailed diff.
6. **Grade & Close** — Must pass: 100% tests, lighthouse ≥96/98/98, Feel Contract intact, zero console errors.
