   - Missing token → 401 Unauthorized
   - Invalid/expired → 401 Unauthorized

<task type="auto">
  <name>Task 2: Create tenant context injection middleware</name>
  <files>rust/raptorflow-api/src/http/middleware/tenant.rs</files>
  <action>
Create Axum middleware that sets PostgreSQL session variable for RLS:

1. Read `TenantContext` from request extensions (set by auth middleware)
2. Use SQLx's `acquire()` to get a connection from the pool
3. Execute `SET LOCAL app.current_org_id = $1` before each query
4. Use a wrapper type or `SqlxMy` extension trait to auto-inject this setting

Alternative approach - use a transaction wrapper:

```rust
pub async fn with_tenant_context<F, T>(
    pool: &PgPool,
    org_id: Uuid,
    f: F
) -> Result<T>
where
    F: FnOnce(&mut PgConnection) -> Result<T>,
{
    let mut tx = pool.begin().await?;
    tx.execute(sqlx::query("SET LOCAL app.current_org_id = $1").bind(org_id)).await?;
    let result = f(&mut *tx).await?;
    tx.commit().await?;
    result
}
```

5. Ensure this is called on EVERY database operation in the service layer

Do NOT rely on application-level filtering - RLS is the source of truth.
</action>

   - `users (id UUID PK, clerk_user_id TEXT UNIQUE, email, name, org_id UUID FK, created_at, updated_at)`

2. Enable RLS on all tenant tables:

   ```sql
   ALTER TABLE organizations ENABLE ROW LEVEL SECURITY;
   ALTER TABLE users ENABLE ROW LEVEL SECURITY;
   ```

3. Create RLS policies:

   ```sql
   CREATE POLICY tenant_isolation ON organizations
       FOR ALL USING (id = current_setting('app.current_org_id', true)::uuid);

       FOR ALL USING (org_id = current_setting('app.current_org_id', true)::uuid);
   ```

4. Create indexes for Clerk ID lookups:

   ```sql
   CREATE UNIQUE INDEX idx_users_clerk_user_id ON users(clerk_user_id);
   CREATE UNIQUE INDEX idx_orgs_clerk_org_id ON organizations(clerk_org_id);
   ```

5. Add to sqlx-data.json for compile-time checking

6. Document the RLS pattern - ALL new tenant tables MUST follow this pattern

Add these queries to sqlx-data.json with proper parameter typing (use $1, $2, etc. for PostgreSQL).

   ```rust
   let signature = request.headers()["clerk-signature"];
   let body = bytes.to_bytes();
   verify_webhook_signature(&body, signature, &CLERK_WEBHOOK_SECRET)?
   ```

3. Parse Clerk event envelope:

   ```json
   {
     "type": "user.created",
     "data": { ... }
   }
   ```

4. Handle events:

   - `user.created`: Insert into `users` table
   - `user.updated`: Update `users` by `clerk_user_id`
   - `organization.created`: Insert into `organizations`
   - `organizationMembership.created`: Link user to org

5. Use idempotency - Clerk retries failed webhooks:

   ```sql
   INSERT INTO ... ON CONFLICT (clerk_user_id) DO UPDATE SET ...
   ```

6. Return 200 quickly - don't do heavy processing in webhook handler
   For slow operations, enqueue to DragonflyDB/SQS

Implementation: Use `serde` to parse JSON, `hmac` crate for signature verification.
