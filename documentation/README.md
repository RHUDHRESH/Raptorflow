- `documentation/module-boundaries.md`: module ownership and dependency rules.
- `documentation/bcm-onboarding-langgraph.md`: BCM-first onboarding contract, flow, and gates.
- `documentation/tooling-catalog.md`: operational tools, scripts, and commands.
- `documentation/reset-roadmap.md`: staged rebuild plan after demolition.
- `documentation/phase3-implementation-backlog.md`: strict dependency-ordered rebuild backlog.
- `documentation/phase3-ticket-breakdown.md`: issue-sized `P3-00xA/B` execution tickets with touch-file lists.
- `documentation/storage-migration.md`: storage system status and migration notes.
- `documentation/root-file-policy.md`: strict root file grouping and prune policy.
- `documentation/generated/`: generated inventories and route maps.
- `documentation/contracts/`: rebuild contracts (API and frontend state maps).

## Rule

All new docs must live under `documentation/` only.
